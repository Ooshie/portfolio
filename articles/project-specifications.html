Project Specifications

April 6, 2022

My team of Luis, Pablo, and I (officially named !FALSE WEB APPS ENTERPRISE) have identified specifications for our web application project. Our idea is to create a website for use by restaurants to handle reservations as well as provide information to customers. Here is how we think our website will be used.

User Stories
  • Guest users will first see our home page where there will be different options to choose from. The users will be able to access pages such as a Menu, Make a Reservation, Contact Information, and an About page about the restaurant.
  • Regular users will be able to sign in to their account and make reservations and see their history.
  • Administrative users will be able to access their customersâ€™ reservations and contact information.
Front-end
  • We are planning on using an API for our Making a Reservation feature.
  • The administrator will be able to access all of the reservations, cancel and modify reservations, open and close available spots, and delete accounts.
  • Users will be able to go to a Menu, Make a Reservation, Contact Information, and information about the restaurant.
Back-end
  • Login - Users will be able to look at past and active reservations. It will also make it easier for users because all their information will be saved.
  • Register - Will allow users to create an account and send their information to the database
  • Logout - exit user.
Database
  • The database will be responsible for registering and signing in users.
  • It will store their information such as their name, email and phone number as well as their reservations.
  • It will also handle administration accounts.
